# API Documentation (Summary)

> This is a reader-friendly endpoint reference for submission/evaluation. The full contract — exact request/response JSON shapes, every field, every error case — is [`docs/API_Contract.md`](../API_Contract.md). If anything here and that document ever appear to disagree, `docs/API_Contract.md` is correct; this summary is generated by re-reading the actual router source, so it reflects the current, as-built endpoint surface even where `API_Contract.md` predates a later addition.

## Conventions

- All endpoints are versioned under **`/api/v1`**.
- Authentication: `Authorization: Bearer <access_token>` on every endpoint except `POST /auth/login`.
- All request/response bodies are JSON, validated by Pydantic schemas.
- Errors share one consistent shape and status-code convention: `401` (missing/invalid/expired token), `403` (valid token, wrong role or ownership), `404` (not found, or hidden to avoid revealing another user's resource), `409` (business-rule conflict, e.g. duplicate attendance), `422` (validation failure).
- **82 endpoints** total across 11 routers.

## 10. Reference Data (`/departments`, `/courses`, `/rooms`, `/semesters`) — 20 endpoints

Each of the four entities exposes the same shape: `GET` (list, any authenticated role), `POST` (create, Admin), `GET /{id}` (Admin, Teacher, Student, Parent), `PUT /{id}` (update, Admin), `DELETE /{id}` (Admin, rejected with `409` if still referenced elsewhere). Course additionally supports filtering the list by `department_id`.
