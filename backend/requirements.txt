# --- On starlette==1.3.1 ---------------------------------------------------

# assumed:
#   - fastapi==0.139.0's own pyproject.toml declares `starlette>=0.46.0`

#   - FastAPI's release notes show dependabot bumping their own pinned test
#     dependency 0.52.1 -> 1.0.0 -> 1.1.0 -> 1.2.1 -> 1.3.1 across merged,
#     CI-passing PRs — i.e. FastAPI's maintainers are testing against and
#     accepting Starlette 1.3.1 right now, not merely tolerating it via an
#     unbounded version range.

# but pip's own resolver (not a hand-picked version) produced this pin both

# is its final release) and incompatible with modern bcrypt — it probes

# 4.1+ removed, crashing on the first hash call. app/core/security.py
